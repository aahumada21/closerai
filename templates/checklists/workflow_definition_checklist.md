# Checklist — Definición de workflow

- Propósito claro (1–2 líneas)
- Input/Output con ejemplo JSON
- Nodos listados y orden lógico
- Idempotencia definida (key + verificación + corte)
- Persistencia definida (tablas/entidades)
- Manejo de errores (retry/corte/handoff)
- Auditoría/observabilidad (trace_id + reason codes)
- Casos de prueba mínimos (3–10)
